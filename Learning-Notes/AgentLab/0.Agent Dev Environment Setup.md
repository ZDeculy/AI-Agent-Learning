## 3. 理解了隐藏文件和配置文件

我们讲了 `.xxx` 文件，例如：

```text
.git
.gitignore
.ssh
.config
.tmux.conf
.env
.venv
.python-version
```

你理解了：

* 以 `.` 开头的文件/文件夹在 macOS/Linux 中默认隐藏；
* `.git` 是 Git 仓库的核心数据库；
* `.gitignore` 是 Git 忽略规则；
* `.ssh` 是用户级 SSH 密钥目录；
* `.env` 用来保存 API key，不能提交 GitHub；
* `.venv` 是 Python 项目虚拟环境，不应该提交 GitHub；
* `.tmux.conf` 是 tmux 配置文件；
* `.config` 常用于放各种软件配置。

## 4. 完整跑通了 Git 本地版本管理

你已经掌握了 Git 的基础流程：

```bash

```

你理解了 Git 的三个区域：

```text
Working Directory 工作区

  ↓ git commit
Local Repository 本地仓库
  ↓ git push
Remote Repository GitHub 远程仓库
```

你也理解了：

* `git status` 查看当前改动；
* `modified` 表示文件相对于上一次 commit 发生变化；

* `git commit -m` 生成本地版本快照；
* `-m` 是 `message`，不是 moment；
* `git push` 把本地 commit 推送到 GitHub。

## 6. 理解并配置了 SSH 认证逻辑

你理解了：

> SSH key 是这台 Mac / 当前用户的，不是某个项目文件夹的。

也就是：

```text
~/.ssh/id_ed25519      私钥，留在本地，不能泄露
~/.ssh/id_ed25519.pub  公钥，放到 GitHub
```

你理解了 SSH 的身份验证逻辑：

```text
Mac 本地保存私钥
GitHub 保存公钥
push / pull 时 GitHub 验证这台电脑是否有匹配私钥
```

你也理解了 HTTPS 和 SSH 都是 Git 与 GitHub 通信的方式：

```text
HTTPS：偏 token / 登录认证
SSH：偏密钥认证，适合长期开发
```

## 8. 理解了 conda、venv、uv、Docker 的关系

我们梳理了：

```text
venv：Python 自带虚拟环境工具
conda：环境管理 + 包管理，适合科学计算/深度学习
uv：现代 Python 项目管理工具，快、轻量、项目驱动
Docker：系统级容器，封装完整运行环境
```

你理解了：

* conda 是“环境驱动”：

  ```text
  conda create
  → conda activate
  → pip/conda install
  → python main.py
  ```

* uv 是“项目驱动”：

  ```text
  uv init
  → uv add
  → uv run
  ```

* Docker 是“容器驱动”，更偏部署和环境复现。

## 10. 理解了 `pyproject.toml` 和 `uv.lock`

你已经理解：

```text
pyproject.toml = 项目说明书
uv.lock = 精确依赖版本锁定表
.venv = 依赖真正安装的位置
```

比如：

```text
uv add openai
```

不仅会安装 openai，还会：

```text
写入 pyproject.toml
解析完整依赖树
写入 uv.lock
安装到 .venv
```

所以 uv 比单纯 `pip install` 更适合现代项目管理。
